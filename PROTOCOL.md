# Call of Core — Protocol Specification

This document defines the two watcher protocols in detail.
For setup instructions see README.md.

---

## Protocol 1: `mp` — MemoryPlugin

Trigger: a fenced code block tagged `mp` anywhere in clipboard text.

```
```mp
{ JSON command object }
```
```

The watcher fires after a 2-second cancellable countdown.
Copying anything else within 2 seconds cancels the operation.
Results are placed back on the clipboard as plain text.

### Commands

#### search
```json
{
  "action": "search",
  "query": "string",
  "count": 10,
  "bucketId": null
}
```
Returns: numbered list of matching memory entries.

#### store
```json
{
  "action": "store",
  "text": "string to remember",
  "bucketId": null
}
```
Returns: confirmation string.
Note: entries are automatically date-stamped with today's UTC date.

#### get
```json
{
  "action": "get",
  "latest": true,
  "count": 10,
  "bucketId": null
}
```
Or `"all": true` to retrieve all entries.
Returns: numbered list of memory entries.

#### get_buckets
```json
{
  "action": "get_buckets"
}
```
Returns: list of buckets with id, name, count, description.

#### recall_chat
```json
{
  "action": "recall_chat",
  "query": "string"
}
```
Returns: summary of relevant past chat history from MemoryPlugin.

### Critical constraint
The MemoryPlugin browser extension uses its own trigger string.
That string must NEVER appear in AI reasoning or response prose —
only inside a properly fenced `mp` block.
Writing it in plain text caused runaway memory writes during testing.

## Protocol 2: `ua` — Understand-Anything

Trigger: a fenced code block tagged `ua` anywhere in clipboard text.

```
```ua
{ JSON command object }
```
```

Requires a `knowledge-graph.json` generated by the Understand-Anything
pipeline. Auto-detected in common locations; use `"path"` to override.

Auto-detection order:
1. `path` field if provided
2. Current working directory `/.understand-anything/knowledge-graph.json`
3. `C:\Users\<user>\genesis-pantheon\.understand-anything\knowledge-graph.json`
4. `C:\m\.understand-anything\knowledge-graph.json`
5. Desktop `\.understand-anything\knowledge-graph.json`

### Commands

#### ua_info
```json
{
  "action": "ua_info",
  "path": "optional/project/path"
}
```
Returns: project name, description, languages, analyzed date,
node count, edge count, layer count, tour step count,
node type breakdown, edge type breakdown, layer names.

#### ua_search
```json
{
  "action": "ua_search",
  "query": "authentication",
  "count": 10,
  "path": "optional/project/path"
}
```
Fuzzy multi-field search across: node name, summary, file path,
node id, tags. Returns top N matches with type, id, file, summary, tags.

#### ua_node
```json
{
  "action": "ua_node",
  "id": "file:src/auth.ts",
  "path": "optional/project/path"
}
```
Exact or partial id match. Returns full node detail plus all
incoming and outgoing edges (up to 8 each shown).

#### ua_layer
```json
{
  "action": "ua_layer",
  "name": "API",
  "count": 10,
  "path": "optional/project/path"
}
```
Case-insensitive layer name match. Returns layer description
and all member nodes.

#### ua_tour
```json
{
  "action": "ua_tour",
  "path": "optional/project/path"
}
```
Returns the guided learning tour generated during `/understand`
analysis, ordered by step number with node references.

## Extending the Protocol

The watcher is designed for extension. To add a new protocol:

1. Add a new regex in `mp-watcher.py`:
   ```python
   MY_RE = re.compile(r"```" + "mytag" + r"\s*([\s\S]*?)```", re.IGNORECASE)
   ```

2. Add it to `find_blocks()` so it participates in document-order detection.

3. Write a `dispatch_my()` function following the same pattern as
   `dispatch_mp()` and `dispatch_ua()`.

4. Add the tag routing in `process()`.

Any HTTP API, any CLI tool, any local service — if it has an interface,
it can be a watcher protocol. The AI writes the block. The watcher fires.
The result comes back.
